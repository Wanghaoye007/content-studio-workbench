# PIAS 图片工作台 UI 设计指南针

## 1. 设计定位

### 方向名称

**Soft Industrial Glass / 柔性工业玻璃**

PIAS 不是展示型网站，也不是消费级图片 App。它是面向高频内容生产的节点式工作台，因此视觉升级必须同时满足：长时间使用不疲劳、复杂流程可扫描、操作反馈有手感、品牌与素材始终比 UI 装饰更突出。

### 一句话原则

**让工具像玻璃一样浮在内容之上，让节点像可靠的生产对象一样落在画布之中。**

### 推荐方案与备选

| 方案 | 表现 | 取舍 | 结论 |
| --- | --- | --- | --- |
| A. 克制高级玻璃 | 深色中性画布、分级圆角、局部毛玻璃、柔和弹性反馈 | 兼顾高级感、密度与性能 | **采用** |
| B. 强 Liquid Glass | 高透明、高模糊、明显折射与悬浮 | 展示感强，但长期使用分心，文字对比不稳定 | 不采用 |
| C. 轻量企业圆润 | 实色面板、统一圆角、轻阴影 | 风险最低，但不足以形成新版辨识度 | 仅作为低性能回退 |

## 2. 灵感来源与转译

这套方向参考成熟产品的原则，不复制其皮肤。

- [Apple Human Interface Guidelines: Materials](https://developer.apple.com/design/human-interface-guidelines/materials)：玻璃应主要服务导航、工具和临时控制层，不应覆盖内容层；玻璃使用必须克制，并提供降低透明度的适配。
- [Linear: A calmer interface for a product in motion](https://linear.app/now/behind-the-latest-design-refresh)：高密度产品中，主任务必须占据最强视觉权重，辅助导航应主动后退；减少边框噪声比增加装饰更高级。
- [Microsoft Fluent Acrylic](https://learn.microsoft.com/en-us/windows/apps/design/style/acrylic)：毛玻璃由透明度、模糊、色调、细噪点和边缘分离共同构成；避免多层 Acrylic 相邻或叠加。
- [Windows Motion](https://learn.microsoft.com/en-us/windows/apps/design/signature-experiences/motion)：动效应直接、连贯、上下文相关；相同入口的面板必须使用一致的进入方向、时长和缓动。
- [n8n 工作区结构](https://support.n8n.io/article/how-to-access-the-n-8-n-admin-panel)：节点工具的核心结构是画布、参数面板和执行记录。PIAS 保留这三个稳定区域，不因视觉升级改变操作心智。
- [Motion Spring](https://motion.dev/docs/spring)：弹性只用于短距离位移和状态切换，不用于持续漂浮或大面积装饰。
- [WCAG 2.2 动效指南](https://www.w3.org/WAI/WCAG22/Understanding/animation-from-interactions)：所有非必要位移动效支持 `prefers-reduced-motion`，避免自动循环和视差运动。

## 3. 六条设计原则

### 3.1 Canvas First

画布、商品图和节点链路永远是第一视觉层。导航、素材栏、工具栏和任务抽屉降低亮度与对比，只有在激活时上浮。

### 3.2 Glass Is Functional

玻璃只用于悬浮工具栏、参数面板、弹窗、任务抽屉、下拉菜单和拖拽反馈。场景节点、结果节点和大面积内容区使用高不透明表面，保证图片与文字稳定。

### 3.3 Soft Geometry, Clear Hierarchy

圆润不等于所有元素都做成胶囊。尺寸越大、离画布越高，圆角越大；越接近精密控制，圆角越小。状态标签和开关可以使用胶囊形，命令按钮保持清晰的工具形态。

### 3.4 One Accent, Semantic Color

蓝色只表示选中、主操作和可连接状态。绿色、黄色、红色仅表达成功、警告、失败。不要使用紫蓝渐变，不用多彩光晕制造“AI 感”。

### 3.5 Motion Explains State

动效解释“从哪里来、发生了什么、去了哪里”。节点拖拽必须直接跟手；参数面板从对应节点方向出现；任务完成通过一次短促的边缘高光反馈，而不是持续闪烁。

### 3.6 Premium Without Fragility

去掉阴影、模糊或动效后，层级仍然成立。玻璃效果是增强，不是结构本身；低性能、降低透明度和降低动态模式都有可靠实色回退。

## 4. 视觉系统

### 4.1 色彩层级

避免纯黑与单一蓝灰。使用略带冷暖差异的中性层，形成可感知但不过度的空间深度。

| Token | 建议值 | 用途 |
| --- | --- | --- |
| `--canvas-0` | `#080A0D` | 无限画布基础色 |
| `--canvas-1` | `#0D1014` | 侧栏后方与画布局部层 |
| `--surface-solid` | `#15181D` | 节点与长期内容面 |
| `--surface-raised` | `rgba(29, 33, 40, .82)` | 参数面板、任务抽屉 |
| `--surface-glass` | `rgba(24, 28, 34, .62)` | 悬浮工具栏、菜单 |
| `--stroke-soft` | `rgba(255, 255, 255, .08)` | 默认分隔 |
| `--stroke-highlight` | `rgba(255, 255, 255, .16)` | 玻璃上沿与选中边缘 |
| `--text-primary` | `#F5F7FA` | 主文字 |
| `--text-secondary` | `#A8AFB9` | 次文字 |
| `--text-tertiary` | `#727A86` | 弱提示 |
| `--accent` | `#4C8DFF` | 唯一主强调色 |
| `--success` | `#43C594` | 成功 |
| `--warning` | `#F0B45A` | 警告 |
| `--danger` | `#F06A6A` | 错误与危险操作 |

背景允许极弱的冷暖径向光差来增强玻璃可见性，但不得出现独立光球、紫色渐变或抢眼装饰。

### 4.2 材质层级

| 层级 | 材质 | 规则 |
| --- | --- | --- |
| L0 画布 | 实色 + 低对比网格 | 不模糊，不发光 |
| L1 内容 | 94%-98% 不透明表面 | 节点、素材项、结果内容，保证图片和文本稳定 |
| L2 常驻工具 | 72%-84% 不透明玻璃 | 导航、素材栏、顶部状态栏，模糊 14-18px |
| L3 临时工具 | 56%-72% 不透明玻璃 | 参数面板、菜单、任务抽屉，模糊 18-24px |
| L4 阻断层 | 高不透明玻璃 + 遮罩 | 弹窗和审核确认，背景降低亮度而非强模糊 |

标准玻璃配方：背景色调 + `backdrop-filter: blur(...) saturate(115%)` + 1px 半透明边缘 + 上沿内高光 + 低透明噪点。禁止玻璃套玻璃，禁止相邻面板使用不同模糊强度形成接缝。

### 4.3 圆角体系

| Token | 尺寸 | 用途 |
| --- | --- | --- |
| `--radius-xs` | 6px | 小型图标按钮、输入框内部控件 |
| `--radius-sm` | 8px | 普通按钮、输入框、分段控制 |
| `--radius-md` | 12px | 节点、素材项、工具组 |
| `--radius-lg` | 16px | 参数面板、任务抽屉、菜单 |
| `--radius-xl` | 20px | 模态弹窗和大型浮层 |
| `--radius-pill` | 999px | 状态标签、开关、进度徽标，仅限语义成立时 |

同一容器内，子元素圆角必须小于父元素至少 4px。图片与节点共用顶部圆角，节点底部信息区不做第二张嵌套卡片。

### 4.4 阴影与边缘

- 常驻面板：`0 12px 30px rgba(0,0,0,.22)`，仅用于和画布分离。
- 临时面板：`0 18px 52px rgba(0,0,0,.36)`，增加 `inset 0 1px rgba(255,255,255,.06)`。
- 节点默认：短而紧的接触阴影；悬停增加边缘亮度，不大幅漂浮。
- 节点选中：1px 主强调边 + 外扩 3px 的低透明蓝色环，不使用霓虹光。
- 禁止四周等强阴影、连续发光和多个彩色阴影叠加。

### 4.5 字体与排版

- 拉丁与数字：`Manrope`，强调现代工具感和清晰数字。
- 中文：`Noto Sans SC`，与 Manrope 使用相近字重。
- 仅使用 400 / 500 / 600 三档字重，避免到处粗体。
- 工作台正文保持 13-14px 的高密度尺度；按钮和关键状态不低于 12px；移动端正文不低于 14px。
- 所有字距为 0。节点标题单行省略，辅助信息最多两行，避免卡片因文字变化跳动。

## 5. 组件方向

### 5.1 主导航

- 导航轨道是低对比长驻玻璃层，不与画布争抢。
- 图标按钮为 40x40px，8-10px 圆角；激活态使用轻亮底和左侧短指示条。
- 悬停只增加背景亮度和图标对比，不整体放大。

### 5.2 顶部状态栏

- 作为单层横向玻璃，不拆成多个小卡片。
- 项目名称权重最高；自动保存、点数和任务数量是弱状态组。
- 状态变化采用数字淡入和一次 120ms 高光，不做滚动数字秀。

### 5.3 场景与素材栏

- 常驻但低亮，激活素材的缩略图和文字获得局部高光。
- 素材项是可拖拽对象，因此允许 10-12px 圆角和轻微按压反馈。
- 分组标题不使用卡片包裹，只通过间距和弱分隔组织。

### 5.4 画布工具栏

- 采用最明显的“清透玻璃”，因为它浮在视觉内容上且文字很少。
- 图标按钮固定尺寸，不因激活或 Tooltip 改变布局。
- 激活工具使用实体强调色底；未激活工具保持透明，避免一排发光按钮。

### 5.5 场景、任务与结果节点

- 节点属于内容层，不做高透明玻璃。
- 场景节点最大、任务节点更紧凑、结果节点更轻，继续保持三类形态差异。
- 悬停：上移 1px、边缘提亮；选中：上移 2px、强调环出现。
- 拖拽中取消过渡，保证指针与节点一一对应；松手后用短弹性完成吸附。
- 执行中的任务节点允许进度高光沿边缘运行一次，但禁止无限呼吸。

### 5.6 参数面板

- 使用 16px 圆角的常规玻璃，文本区保持较高不透明度。
- 从被操作节点的相反侧进入，180-220ms；关闭沿原方向返回。
- 控件组通过 12-16px 间距组织，不增加嵌套卡片。
- 滑杆、分段选择、九宫格锚点和开关均提供 hover、pressed、selected、disabled 四态。

### 5.7 任务抽屉、菜单与弹窗

- 任务抽屉与顶部状态栏使用相同材质家族，但拥有更高海拔与更强阴影。
- 菜单宽度稳定，打开点与触发器保持空间关系。
- 弹窗使用 20px 圆角、高不透明玻璃和背景暗化；危险操作按钮保留红色语义，不使用红色大面积底。

## 6. 动效指南

### 6.1 时间尺度

| 类型 | 时长 | 缓动 |
| --- | --- | --- |
| Hover / Focus | 90-120ms | `ease-out` |
| Press / Toggle | 80-140ms | 快速回弹，视觉缩放最低 0.97 |
| Tooltip / Menu | 140-180ms | 淡入 + 4px 位移 |
| 参数面板 / 抽屉 | 180-240ms | `cubic-bezier(.2,.8,.2,1)` |
| 节点吸附 / 结果出现 | 240-320ms | 低弹性 Spring，禁止明显来回抖动 |
| 画布聚焦 / Fit View | 260-360ms | 平滑 ease-in-out |

### 6.2 手感规则

- **Hover 是确认可操作，不是表演。** 只改变边缘、亮度和 1px 位移。
- **Pressed 必须立即发生。** 按下 60ms 内出现压缩或亮度变化，释放后回弹。
- **Drag 必须零延迟。** 拖拽时关闭节点阴影和位置过渡，只保留轻微缩放与抓取态。
- **Panel 必须有来源。** 左侧触发的面板从左进入，节点右侧参数从右进入。
- **Success 只庆祝一次。** 结果出现可有一次 300ms 边缘扫光，之后回到静态。
- **Loading 不推动布局。** 骨架、进度条、错误文字都在稳定尺寸内变化。

### 6.3 禁止动效

- 不使用自动循环漂浮、呼吸、旋转或背景流光。
- 不使用大范围视差和跟随鼠标的背景光球。
- 不给所有图标添加独立动画。
- 不在拖拽、缩放和画布平移中使用高成本模糊动画。

## 7. 可访问性与性能

- 所有文本在玻璃上仍满足 WCAG AA 对比；玻璃后方图片过亮时自动提高遮罩不透明度。
- `prefers-reduced-motion: reduce` 下关闭位移、缩放和弹性，仅保留 80-120ms 的颜色或透明度变化。
- `prefers-contrast: more` 或透明度关闭时，所有玻璃回退为 `--surface-solid`，边框提高到 16%-22% 白色透明度。
- 同屏高模糊层控制在 5 个以内；大面积长驻面板优先 14px 模糊，临时小面板才使用 20-24px。
- 动效优先使用 `transform` 和 `opacity`，不动画 `width`、`height`、`box-shadow` 的大范围模糊半径。
- 移动端降低阴影半径、模糊强度和同时显示的浮层数量。

## 8. 明确禁区

1. 不把整个页面做成半透明卡片拼贴。
2. 不在页面章节外再包“浮动卡片”。
3. 不使用紫色、紫蓝渐变或霓虹光作为 AI 身份表达。
4. 不给所有元素相同的大圆角；胶囊仅用于状态和开关。
5. 不使用玻璃套玻璃、卡片套卡片。
6. 不让毛玻璃降低商品图、标签文字和审核状态的可读性。
7. 不因视觉升级改变已有节点拖拽、连接、参数和任务流程。
8. 不使用无限循环动效制造“活力”。

## 9. 实施顺序

### Phase 1：基础设计令牌

建立颜色、材质、圆角、阴影、时长、缓动与降级模式；替换分散的硬编码值。

### Phase 2：工作台骨架

依次升级主导航、顶部状态栏、素材栏、画布工具栏、任务抽屉和参数面板，先形成稳定的空间层级。

### Phase 3：节点与控制手感

统一节点 hover / selected / dragging / running / succeeded 状态，补全按钮、滑杆、分段控制、九宫格锚点和菜单反馈。

### Phase 4：动效与响应式

统一进入方向和时间尺度，增加 reduced-motion、低透明度回退，并在桌面、平板和移动端截图复查。

## 10. 验收标准

- 第一眼焦点落在画布商品与当前节点，而不是导航或毛玻璃效果。
- 截图转为灰度后，画布、节点、工具和临时面板仍有清晰的四级层次。
- 圆角有明确层级，页面中不存在“所有东西都像胶囊”的观感。
- 同一类面板的进入方向、时长和关闭行为一致。
- 按钮按下、节点选中、拖拽开始、任务完成都有可感知且不拖沓的反馈。
- 桌面 1280px 以上保持高密度；390px 移动端无横向溢出、遮挡或文字截断。
- 开启 reduced-motion 后不出现非必要位移或弹性动画。
- 关闭 `backdrop-filter` 后仍可操作、可读且层级成立。
- 功能测试与 Fal 节点执行流程保持通过。

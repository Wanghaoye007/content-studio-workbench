# PIAS 拖线新增节点交互设计

## 1. 目标

补齐图片工作台的节点式新增主流程：用户从已有 Scene 或 Result 节点拖出连接线，在画布空白区域松手后，系统必须在落点附近展示节点类型选择器；用户选择操作后进入参数配置，提交时在原落点创建任务节点并保留连接关系。

本设计不替代现有素材拖放。素材拖入空白画布仍创建源 Scene，素材拖到图片节点仍绑定为融图参考。

## 2. 核心交互

### 2.1 新增入口

- 选中的 Scene 和 Result 节点右侧显示一个蓝色圆形新增手柄。
- 手柄可视尺寸为 36px，点击和拖拽命中区域为 44px。
- 手柄使用加号图标，并提供“拖拽新增节点”中文可读名称和 Tooltip。
- 未选中节点时不显示主新增手柄，保留普通血缘连接点用于连线呈现。

### 2.2 拖拽与松手

1. 用户从新增手柄开始拖拽。
2. 画布显示从源节点到指针的临时连接线。
3. 指针经过可释放空白区域时显示落点提示。
4. 用户在空白区域松手后，不立即写入领域数据。
5. 系统在松手位置附近打开节点类型选择器，并保存源节点与画布坐标。
6. 在节点、工具栏、侧栏或画布外松手时取消本次创建，不弹出选择器。

单击新增手柄时，在源节点右侧的默认位置打开同一个节点类型选择器，作为键盘和精确指针操作的等价路径。

### 2.3 节点类型选择器

选择器使用两列紧凑布局，包含八种图片任务：

- 生成
- 融图
- 快速视角
- 定向光
- 去除
- 抠图
- 扩图
- 超分

每个选项展示图标、中文名称和一行简短用途。选择器宽度为 320px，最大高度不超过当前画布可视区域；靠近右侧或底部松手时自动向左或向上翻转。

按 `Escape`、点击画布空白区域或点击关闭按钮会取消选择。取消不会创建 Scene、Job、Edge、Reserve 或 Audit 数据。

### 2.4 待配置节点

用户选择工具后：

- 节点选择器关闭。
- 在松手位置显示一个 220px × 160px 的待配置任务节点。
- 源节点与待配置节点之间显示蓝色虚线连接。
- 对应工具参数面板贴近待配置节点打开。
- 待配置节点显示工具名称、“待配置”状态和取消按钮，不进入任务队列，不占用点数。

待配置节点属于前端临时交互状态，不写入 `StudioState`。用户关闭参数面板或切换源节点时，临时节点与虚线一并移除。

### 2.5 提交转换

用户提交参数时：

1. 复用待配置节点的画布坐标创建 Job。
2. 创建真实 Scene → Job 或 Result → 派生 Scene → Job 关系。
3. 用真实任务节点替换待配置节点，尺寸和中心点保持稳定。
4. 任务进入队列并执行现有点数预留、进度、结果和审核流程。
5. 失败任务保留在该位置，可重试或查看原因。

任务提交前额度不足或参数校验失败时，待配置节点和参数面板继续保留，用户可修改后重试。

## 3. 状态与组件边界

### 3.1 交互状态

`interactionMachine` 新增以下状态：

- `connecting-node`：连接线拖拽中。
- `choosing-node-type`：已保存源节点和落点，选择器打开。
- `configuring-draft-node`：工具已选，临时节点和参数面板打开。

临时创建上下文包含：

- `sourceNodeId`
- `dropScreenPosition`
- `dropCanvasPosition`
- `selectedTool`
- `menuPlacement`

### 3.2 组件

- `CanvasNodes`：渲染选中节点的大尺寸新增手柄。
- `NodeTypePicker`：渲染八种操作和关闭交互。
- `DraftTaskNode`：渲染待配置节点。
- `graph`：把临时节点和临时边合并到 React Flow 展示图，不写入领域状态。
- `Workbench`：处理 `onConnectStart`、`onConnectEnd`、菜单定位、工具选择和提交转换。
- `domain`：允许 `createJob` 接收明确的任务坐标，派生任务继续遵守现有血缘和审计规则。

## 4. 尺寸与视觉

- 新增手柄：36px 可视尺寸，44px 命中区域。
- 节点类型选择器：320px 宽，两列布局，8px 圆角。
- 待配置节点：220px × 160px，8px 圆角。
- 临时连接线：蓝色、虚线、1.5px，不使用高频动画。
- 菜单和临时节点层级高于普通节点，低于参数面板和确认弹层。
- `prefers-reduced-motion` 下取消菜单位移和节点出现动画，保留透明度变化。

## 5. 异常与边界

- 从 Job 节点不能发起新增分支。
- 从不存在或已删除的源节点结束连接时直接取消。
- 在移动端小于 768px 时不提供该编辑入口。
- 节点类型选择器打开期间再次开始连接，会先取消旧的临时创建上下文。
- 参数面板关闭、源节点变化、画布重置和任务提交都会清理临时状态。
- 重复快速松手只保留最后一次有效连接结束事件。
- 临时节点不进入撤销、审计、用量和任务队列；提交后的真实领域变更遵循现有规则。

## 6. 测试与验收

### 6.1 单元测试

- 从 Scene 和 Result 开始连接时进入 `connecting-node`。
- 空白区域松手后进入 `choosing-node-type` 并保存准确坐标。
- 无效区域松手不打开选择器。
- 选择工具后生成一个临时节点和临时边。
- 取消、切换节点和关闭面板会清理临时状态。
- 提交时 Job 使用临时节点坐标，且不会重复创建任务。

### 6.2 浏览器验收

- 1440×900、1024×768 下新增手柄清晰可拖，命中稳定。
- 从源 Scene 拖到空白区域，松手位置出现八项节点类型选择器。
- 选择“融图”后出现 220px × 160px 待配置节点和融图参数面板。
- 取消不增加 Scene、Job、Edge 或任务队列数量。
- 提交后临时节点原位转换为真实任务节点，连接线不跳动。
- 靠近画布四个边缘松手时，选择器始终完整可见。
- 375×812 移动端不显示新增手柄和节点类型选择器。
- 全流程浏览器控制台无错误，页面无控件重叠或文字溢出。

## 7. 非目标

- 本轮不实现任意节点类型插件系统。
- 本轮不允许用户手工连接两个既有节点改变血缘。
- 本轮不新增多人协作、撤销栈或服务端画布租约。
- 本轮不改变素材拖放、任务计费、审核和结果下载规则。
